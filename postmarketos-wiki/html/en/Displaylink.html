Driver

UDL

udl is kernel module for USB 2.0 DisplayLink devices.It is in the mainline.All postmarket OS's kernel dont have udl kernel module so you need to enable it in the kernel config (DRM_UDL) and recompile the kernel.Also it is quite finicky. Some usb 2 displaylink work with it, some dont.(Only HP NL571AA is tested to work.) If your USB 2.0 DisplayLink devices with udl isnt work with udl (For example : evga ev16+), your will get this error from dmesg. 

[  228.747008] [drm] vendor descriptor length:e0 data:00 00 00 00 00 00 00 00 00 00 00
[  228.747032] [drm:udl_init [udl]] *ERROR* Unrecognized vendor firmware descriptor
[  228.749618] [drm] Initialized udl 0.0.1 20120220 for 1-1.3:1.0 on minor 1
[  228.749637] [drm] Initialized udl on minor 1
[  228.885058] udl 1-1.3:1.0: [drm] fb1: udldrmfb frame buffer device
[  228.885358] usbcore: registered new interface driver udl

EDVI

edvi is a out of tree kernel module for USB 3 DisplayLink devices.It require userland components to work.
