Vendors

Samsung

Samsung has used Tizen in their modern TVs. 2012-2014 models used the (retroactively renamed) "Samsung Legacy Platform", see here .

Samsung explicitly writes in the software license information that they only provide the source code for the TV if it is requested by someone at the latest three years after they shipped the last batch of the model. If you have a Samsung TV that is only a few years old and the source is not already published you might want to request the source code now , in case you want to look at it in the future.

The user SkyBladeCloud has used the Ubuntu userspace on the 46ES8000, so it seems that the legacy platform is also Linux based. See here .

See also the SamyGo project and this LWN article about it.

LG

LG used Netcast in their 2010-2014 models and WebOS (which uses OpenEmbedded) since then.

LG is publishing open source code on https://opensource.lge.com

According to an OpenLGTV developer, there are no TVs with mainline support, because there are a lot of hardware customizations. Booting a custom kernel requires hacking the bootloader which is normally verifying signatures.

Philips

Philips used NetTV on older models and Android TV on newer TVs.

https://fredericb.info/2014/11/exploitation-of-philips-smart-tv.html

The Android TV models can be rooted, except when dm-verify is enabled: https://forum.xda-developers.com/t/guide-how-to-root-2015-philips-android-tv.3323912/

Possibility for booting into other kernels using kexec.

Sony

Sony Brava TVs run Linux

Old exploit which has been patched: https://github.com/CFSworks/nimue
